链表

发表于 2022-05-23 更新于 2022-05-30 分类于 算法与数据结构
什么是链表？链表是一种通过指针串联在一起的线性结构，每一个节点由两部分组成，一个是数据域一个是指针域（存放指向下一个节点的指针），最后一个节点的指针域指向null（空指针的意思）。

基础及定义

链表的类型

  • 单链表：一种通过指针串联在一起的线性结构，每一个节点由两部分组成，一个是数据域一个是指针域（存放指向下一个节点的指针），最后一个节点的指针域指向null（空指针的意思）。
  • 双链表：每一个节点有两个指针域，一个指向下一个节点，一个指向上一个节点，双链表既可以向前查询也可以向后查询。
  • 循环链表：链表首位相连，可以来解决约瑟夫环问题。

链表的存储方式

  • 数组是在内存中是连续分布的，但是链表在内存中可不是连续分布的。
  • 链表是通过指针域的指针链接在内存中各个节点。
  • 链表中的节点在内存中不是连续分布的 ，而是散乱分布在内存中的某地址上，分配机制取决于操作系统的内存管理。

相关题目

反转链表（206）

  • 问题描述：给你单链表的头节点 head，请你反转链表，并返回反转后的链表。

  • 迭代：定义一个 pre 指针指向 null，cur 指针指向 head，在遍历链表时，将当前节点的 next 指针改为指向前一个节点。由于节点没有引用其前一个节点，因此必须事先存储其前一个节点。在更改引用之前，还需要存储后一个节点。最后返回新的头引用。

  • 递归：以链表 1 -> 2 -> 3 -> 4 -> 5 举例

反转链表Ⅱ（92）

  • 问题描述：给你单链表的头指针 head 和两个整数 left 和 right ，其中 left <= right 。请你反转从位置 left 到位置 right 的链表节点，返回反转后的链表 。

  • 设置 node 指针指向第 left - 1个节点，与反转链表类似，pre 指向 node，cur 指向 node.next，循环 right - left 次，最终 pre 指向第 right 个节点，cur 指向第 right + 1 个节点，接着将 node.next.next 和 cur 连接，node.next 和 pre连接，将整个链表串联起来，返回 dummyNode.next 节点。

K 个一组反转链表（25）

  • 问题描述：给你链表的头节点 head ，每 k 个节点一组进行翻转，请你返回修改后的链表。

    k 是一个正整数，它的值小于或等于链表的长度。如果节点总数不是 k 的整数倍，那么请将最后剩余的节点保持原有顺序。

    你不能只是单纯的改变节点内部的值，而是需要实际进行节点交换。

    1. 利用 count 在每次循环时计数，当某次循环 count < k 时，说明已经到了末尾，退出循环；
    2. 在 start 。。。。end 范围里的节点进行翻转（不包括 start 和 end 节点），参考反转链表Ⅱ；
    3. 每次反转前保存 start.next 节点，因为经过反转后它就来到了下次反转的 start 节点处

合并两个有序链表（21）

  • 题目描述：将两个升序链表合并为一个新的 升序 链表并返回。新链表是通过拼接给定的两个链表的所有节点组成的。

  • 类似归并排序，通过比较每个链表节点的值，小的加到 dummyNode 上，如果有一条链表为空了，则将另一条链表直接接在末尾。

合并 K 个升序链表（23）

  • 题目描述：给你一个链表数组，每个链表都已经按升序排列。请你将所有链表合并到一个升序链表中，返回合并后的链表。

  • 利用堆，重写优先队列，根链表节点值大小进行排序，即堆顶的节点值最小，首先将每个链表的头节点入堆，其次弹出堆顶，加入新链表，直至堆为空

  • 分治法，即归并排序的链表形式 TODO

环形链表（141）

  • 题目描述：给你一个链表的头节点 head ，判断链表中是否有环。如果链表中有某个节点，可以通过连续跟踪 next 指针再次到达，则链表中存在环。

  • 快慢指针，慢指针走一步，快指针走两步，若两指针相遇，则说明链表有环，若快指针走到空了，则说明链表没环。

环形链表Ⅱ（142）

  • 题目描述：给定一个链表的头节点 head，返回链表开始入环的第一个节点。 如果链表无环，则返回 null。如果链表中有某个节点，可以通过连续跟踪 next 指针再次到达，则链表中存在环。不允许修改链表。

  • 先判断链表是否有环，无环直接返回 null，若有环，则定义两个指针，一个指向头节点，一个指向快慢指针相遇的节点，两个指针同速走，相遇点即环的入口。

链表中倒数第 K 个节点（剑指 Offer 22）

  • 题目描述：输入一个链表，输出该链表中倒数第k个节点。为了符合大多数人的习惯，本题从1开始计数，即链表的尾节点是倒数第1个节点。例如，一个链表有 6 个节点，从头节点开始，它们的值依次是 1、2、3、4、5、6。这个链表的倒数第 3 个节点是值为 4 的节点。

  • 快慢指针，快指针先走 k 步，然后快慢指针一起走，当快指针走完时慢指针的位置就是倒数第 k 个节点。

删除链表倒数第 N 个节点（19）

  • 题目描述：给你一个链表，删除链表的倒数第 n 个结点，并且返回链表的头结点。

  • 遇上题类似，利用快慢指针找到倒数第 N+1 个节点，然后删除它的 next 节点。

相交链表（160）

  • 题目描述：给你两个单链表的头节点 headA 和 headB，请你找出并返回两个单链表相交的起始节点。如果两个链表不存在相交节点，返回 null。

  • 定义两个指针分别指向 headA 和 headB，同时前进若走到底则换到另一个链表上，直至相遇，若同时为 null，则不存在相交节点。

两数相加（2）

  • 题目描述：给你两个 非空 的链表，表示两个非负的整数。它们每位数字都是按照 逆序 的方式存储的，并且每个节点只能存储 一位 数字。请你将两个数相加，并以相同形式返回一个表示和的链表。你可以假设除了数字 0 之外，这两个数都不会以 0 开头。

  • 设置一个进位 carry，两条链表每个节点相加，空节点当作 0 来处理，sum / 10 为进位，sum % 10 相加节点的值

两数相加Ⅱ（445）

  • 问题描述：给你两个 非空 链表来代表两个非负整数。数字最高位位于链表开始位置。它们的每个节点只存储一位数字。将这两数相加会返回一个新的链表。

  • 与上题正好反过来，所以使用栈来存储节点的值，注意初始化新节点 head = null，每次循环产生的新节点 node，设置 node 的下一个节点为 head，然后再将 head 指向 node，这样最后返回的 head 即为链表的头节点。

分隔链表（86）

  • 题目描述：给你一个链表的头节点 head 和一个特定值 x ，请你对链表进行分隔，使得所有小于 x 的节点都出现在大于或等于 x 的节点之前。你应当保留两个分区中每个节点的初始相对位置。

  • 创建两条链表，一条为小于 x 的节点，另一条为大于或等于 x 的节点，然后再合并。

排序链表（148）

  • 题目描述：给你链表的头结点 head，请将其按 升序 排列并返回 排序后的链表 。

  • 堆排序，所有节点入堆，在弹出堆顶元素连接到新的头节点后面，注意排序完最后一个节点要指向 null。

  • 归并排序

回文链表（234）

  • 题目描述：给你一个单链表的头节点 head ，请你判断该链表是否为回文链表。如果是，返回 true ；否则，返回 false 。

  • 先找到链表的中间节点，反转右半部分链表，判断左右这两部分链表是否相等，若相等则再次反转右半部分，恢复至初始。
