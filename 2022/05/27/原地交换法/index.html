原地交换法

发表于 2022-05-27 分类于 算法与数据结构
当遇到只能使用常数级别空间的题目时，并且要找的数一定在 [1, N + 1] 的区间里，可以考虑把原始的数组当成哈希表来使用。

相关题目

数组中重复的数字（剑指 Offer 03）

  • 题目描述：在一个长度为 n 的数组 nums 里的所有数字都在 0～n-1 的范围内。数组中某些数字是重复的，但不知道有几个数字重复了，也不知道每个数字重复了几次，请找出数组中任意一个重复的数字。

  • 哈希法、排序后遍历

  • 原地交换法，遍历数组 nums，设索引初始值 i = 0；若 nums[i] == i，说明此数字已在对应索引位置，无需交换，因此跳过；若 nums[nums[i]] == nums[i]，代表索引 nums[i] 处和索引 i 处的元素值都为 nums[i]，即找到了重复值，返回 nums[i]；否则，交换索引为 i 和 索引为 nums[i] 的元素值
