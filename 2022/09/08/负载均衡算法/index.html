负载均衡算法

发表于 2022-09-08 更新于 2022-09-19 分类于 微服务
在写 RPC 启动项的时候遇到了负载均衡算法的问题，因此在这系统性地整理一下这些算法。

负载均衡概述

负载均衡（Load Balance），意思是将负载（如前端的访问请求）进行平衡、（通过负载均衡算法）分摊到多个操作单元（服务器，中间件）上进行执行。是解决高性能，单点故障（高可用），扩展性（水平伸缩）的终极解决方案。可以理解为，负载均衡是高可用和高并发共同使用的一种技术。

负载均衡的作用

  1. 增加吞吐量，解决并发压力（高性能）
  2. 提供故障转移（高可用）
  3. 通过添加或减少服务器数量，提供网站伸缩性（扩展性）
  4. 安全防护（负载均衡设备上做一些过滤，黑白名单等处理）

一些常见的负载均衡

Nginx

  • 轮询
  • 加权轮询
  • IP 哈希
    • 依据发出请求的客户端 IP 的 hash 值来分配服务器，该算法可以保证同 IP 发出的请求映射到同一服务器，或者具有相同 hash 值的不同 IP 映射到同一服务器，在一定程度上解决了集群部署环境下 Session 不共享的问题
  • URL hash
    • 根据请求的 URL 的 hash 值来分配服务器。该算法的特点是，相同 URL 的请求会分配给固定的服务器，当存在缓存的时候，效率一般较高
  • 最小连接数
    • 当有新的请求出现时，遍历服务器节点列表并选取其中连接数最小的一台服务器来响应当前请求。连接数可以理解为当前处理的请求数

Dubbo

  • 加权随机
  • 轮询
  • 最少活跃调用数
    • 如果每个提供者的活跃数相同，则随机选择一个
    • 在每个服务提供者里维护着一个活跃数计数器，用来记录当前同时处理请求的个数，也就是并发处理任务的个数。这个值越小，说明当前服务提供者处理的速度越快或者当前机器的负载比较低，所以路由选择时就选择该活跃度最小的机器
  • 一致性 Hash
    • 相同参数的请求总是发到同一提供者，当某一台提供者机器宕机时，原本发往该提供者的请求，将基于虚拟节点平摊给其他提供者，这样就不会引起剧烈变动

Feign（集成了 Ribbon，主要是通过 Ribbon 来实现负载均衡）

  • 轮询
  • 随机
  • 最小连接
  • 加权
  • 重试

优点：服务器请求数目相同；实现简单、高效；易水平扩展

缺点：服务器压力不一样，不适合服务器配置不同的情况；请求到目的结点的不确定，造成其无法适用于有写操作的场景

加权随机

由于完全随机算法下每个服务器的访问概率大致相同，所以如果部分服务器性能不一致的话，容易导致性能差的服务器压力过大，所以要根据服务器性能不一致的情况，给性能好的服务器多处理请求，给差的少分配请求（能者多劳），所以就需要在随机算法的基础上给每台服务器设置权重，延伸为加权随机算法

在依据权重找对应的服务时，随机数 - 权重值，如果小于 0，说明次数在权重值的范围内，如果不小于 0，说明没在这个区间内，就要往后面去找

轮询

按顺序依次调度每个服务器，轮询算法与服务器权重没有关系，每个服务器都会有序地被轮询到

测试结果

优点：服务器请求数目相同；实现简单、高效；易水平扩展

缺点：服务器压力不一样，不适合服务器配置不同的情况；请求到目的结点的不确定，造成其无法适用于有写操作的场景

加权轮询

与加权随机类似，根据权重来轮询请求服务

测试结果

优点：可以将不同机器的性能问题纳入到考量范围，集群性能最优最大化

缺点：生产环境复杂多变，服务器抗压能力也无法精确估算，静态算法导致无法实时动态调整节点权重，只能粗糙优化；同时根据测试结果来看，该算法先轮询 A 服务器，再轮询 B 服务器，再轮询 C 服务器，所以他在一段时间内去请求的服务器永远是一个，当某个服务器权重过大会造成该服务器压力比较大

平滑加权轮询

平滑加权轮询会把轮询的请求 打散，让每个服务器都比较均衡地获得请求，避免了同一个时刻大量请求打入，使请求更加 平衡

假设有 N 台服务器 S = {S0, S1, S2, …, Sn}，默认权重为 W = {W0, W1, W2, …, Wn}，当前权重为 CW = {CW0, CW1, CW2, …, CWn}。在该算法中有两个权重，默认权重表示服务器的原始权重，当前权重表示每次访问后重新计算的权重，当前权重的出初始值为默认权重值，当前权重值最大的服务器为 maxWeightService，所有默认权重之和为 weightSum，服务器列表为 list，算法可以描述为：

  1. 找出当前权重值最大的服务器 maxWeightService；
  2. 计算 {W0, W1, W2, …, Wn} 之和 weightSum；
  3. 将 maxWeightService.CW = maxWeightService.CW - weightSum；
  4. 重新计算 {S0, S1, S2, …, Sn} 的当前权重 CW，计算公式为 Sn.CW = Sn.CW + Sn.Wn
  5. 返回 maxWeightService
